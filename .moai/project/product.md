## 미션

Composify는 React 애플리케이션에 비주얼 에디터를 추가하는 오픈소스 라이브러리입니다. 개발자가 아닌 팀원들도 기존 프로덕션 컴포넌트를 사용하여 페이지를 구성할 수 있게 하여, 엔지니어가 실제 기능 개발에 집중할 수 있도록 합니다.

---

## 비전

**Server Driven UI를 모든 팀에게**

대부분의 비주얼 빌더는 이분법적 선택을 강요합니다: 일반적인 컴포넌트를 가진 경직된 페이지 빌더(Wix, Squarespace)를 사용하거나, 플랫폼에 맞게 코드를 수정해야 하는 복잡한 헤드리스 CMS(Builder.io, Puck, Storyblok)를 도입해야 합니다.

Composify는 그 중간에 위치합니다. 실제 컴포넌트 코드를 위한 비주얼 인터페이스입니다. 컴포넌트를 한 번 등록하면 팀의 누구나 비주얼하게 페이지를 구성할 수 있습니다.

---

## 핵심 가치 제안

### 1. 배포 없이 UI 변경 가능
대부분의 앱은 하드코딩된 UI로 배포됩니다. 작은 수정도 전체 재배포가 필요합니다. Composify를 사용하면 UI가 서버에 있어 즉시 변경하고 반영할 수 있습니다. CI/CD 없이, 앱스토어 리뷰 없이, 대기 없이.

### 2. 팀 생산성 향상
엔지니어가 잘하는 일에 집중: 견고하고 재사용 가능한 컴포넌트 구축
비개발자가 비주얼 에디터로 컴포넌트 활용하여 페이지 구성

### 3. A/B 테스팅과 빠른 프로토타이핑
사용자 세그먼트별로 다른 페이지 레이아웃을 테스트하거나 프로덕션에서 직접 기능을 프로토타입할 수 있습니다. 페이지 구조 교체, 온보딩 플로우 개인화, 새 CTA 테스트 모두 서버 사이드에서 이루어집니다.

### 4. 헤드리스 CMS보다 나은 대안
전통적인 CMS는 테마와 템플릿에 가둡니다. Composify를 사용하면 콘텐츠 팀이 핵심 제품과 동일한 컴포넌트를 사용합니다. 편집한 것이 정확히 배포됩니다.

## 타겟 사용자

### 주요 사용자 그룹

1. **프론트엔드 개발자**
   - 역할: 재사용 가능한 React 컴포넌트 구축 및 Catalog 등록
   - 니즈: 디자인 시스템 유지, 코드 품질 보장, 비개발자 요청 최소화
   - 가치: 컴포넌트 한 번 등록으로 반복 작업 제거

2. **콘텐츠 팀 / 마케팅**
   - 역할: 랜딩 페이지, 프로모션 페이지, 캠페인 콘텐츠 제작
   - 니즈: 개발자 의존 없이 빠른 페이지 수정
   - 가치: 비주얼 에디터로 즉시 페이지 구성 및 발행

3. **제품 관리자**
   - 역할: A/B 테스트, 사용자 세그먼트별 UI 실험
   - 니즈: 배포 사이클 없이 빠른 실험
   - 가치: 서버 사이드 UI 변경으로 즉각적인 실험

4. **디자이너**
   - 역할: 디자인 시스템 컴포넌트 활용 및 페이지 구성
   - 니즈: 실제 프로덕션 컴포넌트로 작업
   - 가치: 개발 결과물과 동일한 컴포넌트 사용

## 핵심 기능

### Editor
- 드래그 앤 드롭 컴포넌트 배치
- 사이드바를 통한 Props 수정
- 요소 중첩 지원
- JSX 문자열로 직렬화
- 코드 에디터 통합 (Monaco Editor)
- 멀티 뷰포트 지원

### Renderer
- JSX 문자열을 실제 React 컴포넌트로 렌더링
- Catalog에 등록된 컴포넌트만 렌더링
- 서버 사이드 렌더링 지원

### Catalog
- 컴포넌트 등록 시스템
- Props 스키마 정의 (type, label, default, options)
- 다양한 속성 타입 지원: string, number, boolean, radio, select, array, object, node, custom

## 경쟁 우위

| 특성 | Composify | 페이지 빌더 (Wix 등) | 헤드리스 CMS |
|------|-----------|---------------------|--------------|
| 실제 프로덕션 컴포넌트 사용 | O | X | 일부 |
| 디자인 시스템 유지 | O | X | 일부 |
| 코드 소유권 | O | X | O |
| 배포 없는 UI 변경 | O | O | O |
| 설치 복잡도 | 낮음 | 해당없음 | 높음 |
| 벤더 종속성 | 없음 | 높음 | 중간 |
